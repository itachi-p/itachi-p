# POLICY.md — AIとの作業ルール

> このファイルは新しいチャットを始める際に毎回AIに読ませる共通仕様書。
> README.mdの「なぜ」に対し、こちらは「どう動くか」を定義する。

## このリポジトリで何をしているか

「仮説設定→検証→判断→記録」のサイクルを構造化データとして蓄積する。
成果物ではなく判断ログが主目的。後でLLMに再投入することを前提とした構造にする。

## AIへの制約

- 感想・道徳的評価・励ましは不要
- 必ず判断（採用/保留/破棄）とその根拠をセットで出力する
- 長文より構造（箇条書き・表・見出し）を優先
- 「検証したつもり」で終わる出力をしない
- ツールや手法の説明より「なぜその判断か」を優先する

## ドキュメント構造ルール

| パス | 内容 | 更新タイミング |
|---|---|---|
| `README.md` | 方針・思想の現在地 | 方針転換時 |
| `POLICY.md` | AIへの指示仕様（このファイル） | ルール変更時 |
| `handover.md` | 次セッションへの申し送り | 毎セッション末 |
| `docs/YYYY-MM-DD_*.md` | 判断・思考ログ | 随時 |
| `projects/YYYY-MM-DD_*/` | 検証プロジェクトの成果物 | プロジェクト毎 |

## 仮説検証の型

1. 視点設定（顧客 / 現場 / 経営）
2. 不便・違和感を言語化
3. 仮説整理（Why / Who / What）
4. 最小構成で試作（AIに指示）
5. 検証
6. 判断（採用 / 保留 / 破棄）＋根拠を必ず記録
7. 次の仮説へ接続

## 捨てる判断基準（現行版）

- 検証コストが仮説の価値を上回る場合 → 破棄
- より優先度の高い仮説が出現した場合 → 保留
- ※プロジェクト毎に更新すること
