## 入力（初期プロンプト）
しばらく来店が遠のいている整体院の既存顧客に送る、紙のハガキ（DM）をデザインしたいです。

目的は「再来店のきっかけを作ること」と「紹介キャンペーンへの参加を促すこと」です。
紹介者とその友人の両方が初回施術を半額で受けられる内容にします。

ハガキにはQRコードを掲載して、そこからキャンペーンページに誘導します（QRは仮で枠だけでOK）。

ターゲットやトーン、掲載すべき情報、コピー、レイアウトなどを含めて提案してください。
必要な情報があれば先に質問してください。

## 実施結果（中間記録）
上記初期プロンプトに対して右側画面に大量の質問が出た。  
しかし、既に日付変更前だったので、「メインターゲットの年齢層：40〜50代」だけを指定し、それ以外を一任してみた。
その回答が以下
```
現時点の前提（仮定）：
ターゲット：40〜50代（指定）
院名：「いろは整体院」（仮）
トーン：信頼・誠実系（和モダン、落ち着いた紺＋生成り＋朱の差し色）
サイズ：通常ハガキ100×148mm 縦向き、両面
メインコピー：心配り系（「お変わりありませんか」）
院長の手書き風メッセージあり
初回6,000円→3,000円（半額）
バリエーション：3案比較（design_canvasで並べる）
design_canvasに3案並べて比較できる形にします。
```

## 認知変化ログ（最大5件）

### ①
- トリガー：Claude Design初使用時、各機能の意味確認
- 変化前：UIの各機能は試行錯誤で探るしかないと思っていた
- 変化後：まずチャットで機能意図や仮説を確認し、その後に最小限の実操作で検証する方が速い
- 再利用：仕様変化が速いベータ環境では、固定マニュアルより対話的確認を優先する
- 確度：中（今後仕様変更で変わる可能性あり）

### ②
- トリガー：「Tweaks」の役割確認
- 変化前：静的HTMLで十分ではないかと思っていた
- 変化後：少なくとも編集体験や即時プレビューのためにJavaScript/JSXが有効に使われていると理解した
- 再利用：静的成果物と編集体験を分けて考える視点を持つ
- 確度：低〜中（内部実装意図は未確認）

### ③
- トリガー：「Draw」機能の試用
- 変化前：一般的な描画ツールに近いものを想定していた
- 変化後：「Comment」は言語化された指示、「Draw」は視覚的・感覚的な指示入力として機能していると理解した
- 再利用：言語化しにくいUI修正は、まずDrawで方向性を示してからCommentで精緻化する
- 示唆：非言語入力をAIが解釈するUXは今後さらに重要になる可能性
