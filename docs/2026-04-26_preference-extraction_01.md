# 2026-04-26_最新AIによるユーザー自身が言語化できてない嗜好の言語化検証①

# 目的
Claude Design初使用体験を通して
再利用可能な思考ログと意思決定パターンを抽出する

# 注意点
* 成果物の品質評価を主目的にしない
* 「AIすごい」で終わらせない
* ログは“後で使う前提”で圧縮も同時に行う

# 仮説
* 曖昧な指示ほどAIは前提条件を質問で補完する
* その質問が「自分の未言語化領域」を露出させる
* その過程を記録すれば再利用可能な思考テンプレになる

# 観測ポイント　※ログを見る時の“観点”を固定
* AIは何を訊いてきたか
* 「なぜそれを訊いた？」と感じたか
* それに対して自分の認識はどう変わったか
* どの時点でアウトプットの質が跳ねたか

## 実験ログ
「全てを記録する」のではなく、「粒度は固定・量は制限する」
* 整形しすぎる → 発見が消える
* 生データそのまますぎる → 再利用できない
↓　
① 生ログ（完全保存）
* 全プロンプト
* AIの質問
* 自分の反応
👉 これは“証拠”として残す
👉 読みやすさは捨てていい

② 抽出ログ（これが本体。ここに制約をかける）
🎯 粒度の決め方（迷わないようルール化する）
基本単位は「1つの認知変化 = 1レコード」

#### 粒度の最適イメージ
悪い例：
* 「色を変えた」
* 「文章を調整した」

* 「“割引訴求”より“安心訴求”の方が重要と気づいた」
